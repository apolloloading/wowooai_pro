# QD 系统 — 导出操作手册(Playbook)

> 适用页面:增员派单 / 增员办理 / 减员派单 / 减员办理
> 适用按钮:**导出办理信息**(blob 响应,必须用 Hook+curl)、**导出参保信息**(直接下载 ZIP)
> AI 代理执行时:遇到 `guide_index.json` 里 `mode: "hook_curl"` 的导出按钮,**必须**参照本文档,不要直接使用 `action='export'`(会被误判为 async_export)。
>
> ⚠️ 登录账号 / 密码 / 验证码一律由用户在浏览器窗口手动完成,本手册不包含任何凭据。

---

## 一、核心原则

| 原则 | 说明 |
|------|------|
| 工具 | 全程使用 `renliwo_browser`,禁止用 `browser_use` 操作业务页 |
| ref 时效 | 每次页面刷新/Tab 切换后 ref 都会变,操作前必须先 `snapshot` |
| JS 语法 | `evaluate` 的 code 必须用 `(function(){...})()` 包裹,**禁用** `const/let`(老 V8 实例兼容) |
| Session | Demo 环境 Session 较短,中途可能跳回登录页 → 让用户重新登录 |
| 导出分流 | 参保信息(ZIP) → `action='export'` 或 `action='export_hook_curl'`;办理信息(blob) → **必须** `action='export_hook_curl'`,或按本手册手动 Hook+curl |
| 文件目录 | 该系统默认下载到 `~/Desktop`,不要去 `~/Downloads` 找 |

---

## 二、推荐路径(端到端封装版)

工具已内置 `action='export_hook_curl'`,适用于"导出办理信息"等 blob 响应按钮。一次调用完成"注入 hook → 点导出 → 点确认弹窗 → 抓 API → 用 token 下载 → 落盘"全流程。

```
renliwo_browser
action: export_hook_curl
btn_text: 导出办理信息
confirm_btn_text: 确认导出           # 弹窗里的最终确认���钮
api_keyword: exportHandleResult     # 用来过滤捕获到的 API(按 url 子串匹配)
save_to: ~/Desktop
filename: 增员办理信息.xlsx          # 可选,不传则用时间戳
timeout: 30
```

**返回示例**:
```json
{
  "ok": true,
  "file": "/Users/<user>/Desktop/增员办理信息.xlsx",
  "size": "13.1 KB",
  "api": "https://wowo-gateway-uat.renliwo.com/demo-qd-services/increase/exportHandleResult",
  "http_code": 200
}
```

**该 action 失败时**(例如新页面没匹配的 API 关键字)按下面"三、手动 Hook+curl 流程"逐步执行。

## 三、手动 Hook+curl 流程(兜底,所有 blob 导出通用)

### 3.1 注入网络请求拦截器

```
renliwo_browser
action: evaluate
code: (function() {
  window.__capturedUrls = [];
  var origFetch = window.fetch;
  window.fetch = function() {
    var url = arguments[0];
    var opts = arguments[1] || {};
    if (typeof url === 'string') {
      window.__capturedUrls.push({method: 'fetch', url: url, opts: opts});
    }
    return origFetch.apply(this, arguments);
  };
  var origXHROpen = XMLHttpRequest.prototype.open;
  var origXHRSend = XMLHttpRequest.prototype.send;
  XMLHttpRequest.prototype.open = function(method, url) {
    this.__xhrMethod = method;
    this.__xhrUrl = url;
    return origXHROpen.apply(this, arguments);
  };
  XMLHttpRequest.prototype.send = function(body) {
    if (this.__xhrUrl) {
      window.__capturedUrls.push({method: 'xhr', url: this.__xhrUrl, reqMethod: this.__xhrMethod, body: body});
    }
    return origXHRSend.apply(this, arguments);
  };
  return 'Hooks installed';
})()
```

### 3.2 点击导出按钮 + 确认弹窗

```
renliwo_browser
action: snapshot
# 记录"导出办理信息"按钮的 ref
```

```
renliwo_browser
action: click
ref: <导出办理信息按钮 ref>
wait_after: 2
```

```
renliwo_browser
action: snapshot
# 记录"确认导出"按钮 ref;"按办理事项导出"默认已选中
```

```
renliwo_browser
action: click
ref: <确认导出按钮 ref>
wait_after: 10
```

### 3.3 提取捕获的 API 调用

```
renliwo_browser
action: evaluate
code: (function() { return JSON.stringify(window.__capturedUrls || []); })()
```

从返回的 JSON 数组里筛选 `url` 含 `exportHandleResult`(或目标 API 关键字)的那一条,从中读取:
- `url`:完整 API 端点
- `opts.method`:POST
- `opts.headers.Authorization`:`bearer <token>`
- `opts.headers["x-user-name"]`
- `opts.body` 或 `body`:请求体(JSON 字符串)

### 3.4 用 curl 下载 blob 文件

```bash
curl -s -X POST "<上一步抓到的 url>" \
  -H "Authorization: bearer <token>" \
  -H "x-user-name: <从 headers 读到的用户名>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '<上一步抓到的 body>' \
  -o ~/Desktop/<目标文件名>.xlsx \
  -w "\nHTTP Code: %{http_code}\nSize: %{size_download}"
```

**预期**:`HTTP Code: 200`,Size > 0。如返回 401 → token 过期,请用户重新登录后从 3.1 重做。

## 四、参保信息导出(直接下载 ZIP)

参保信息走标准下载事件,直接用工具内置的 `export` 即可:

```
renliwo_browser
action: export
btn_text: 导出参保信息
save_to: ~/Desktop
timeout: 20
```

**预期返回**:
```json
{
  "async_export": false,
  "file": "/Users/<user>/Desktop/参保信息YYYYMMDDHHMMSS.zip",
  "size": "35.2 KB"
}
```

### ZIP 解压(避免中文文件名乱码)

ZIP 内文件名是 GBK 编码(Windows 压缩工具默认),macOS `unzip` 直接解压会乱码。**必须用 Python**:

```bash
python3 -c "
import zipfile, os
z = zipfile.ZipFile('/Users/<user>/Desktop/参保信息XXXXX.zip')
out_dir = '/Users/<user>/Desktop/参保信息_解压'
os.makedirs(out_dir, exist_ok=True)
for info in z.infolist():
    name = info.filename.encode('cp437').decode('gbk')
    info.filename = name
    z.extract(info, path=out_dir)
    print('Extracted:', name)
"
```

## 五、增员派单页面 — 操作前置流程(以"后台处理中 + 后台负责人 + 福利所在地 + 派单日期"筛选为例)

### 5.1 切 Tab(Ant Design Tab 必须用 dispatchEvent)

直接 click ref 经常无效,改用:

```
renliwo_browser
action: evaluate
code: (function() {
  var tabs = document.querySelectorAll('.ant-tabs-tab');
  for (var i = 0; i < tabs.length; i++) {
    if (tabs[i].textContent.indexOf('后台处理中') !== -1) {
      tabs[i].dispatchEvent(new MouseEvent('click', {bubbles: true, cancelable: true, view: window}));
      return 'Dispatched click on: ' + tabs[i].textContent.trim();
    }
  }
  return 'Not found';
})()
```

切换后等 8~10 秒让数据刷新,然后看 `pagination` 字段确认数据量变化。

### 5.2 展开筛选

```
renliwo_browser
action: click
selector: text=展开
wait_after: 2
```

### 5.3 后台负责人(普通 input,直接 type)

```
renliwo_browser
action: type
ref: <后台负责人 input ref>
text: <人名>
```

### 5.4 福利所在地(Ant TreeSelect + 虚拟列表)

```
renliwo_browser
action: click
ref: <福利所在地下拉框 ref>
wait_after: 2
```

```
renliwo_browser
action: evaluate
code: (function() {
  var holder = document.querySelector('.ant-select-tree-list-holder');
  if (holder) { holder.style.overflowY = 'auto'; holder.scrollTop = 150; }
  setTimeout(function() {
    var titles = document.querySelectorAll('.ant-select-tree-title');
    for (var i = 0; i < titles.length; i++) {
      if (titles[i].textContent === '<省份名,如:安徽省>') { titles[i].click(); return; }
    }
  }, 500);
  return 'Scrolling...';
})()
```

设置完后**先等 2 秒**再操作日期,否则焦点转移会清空刚选的值。

### 5.5 派单日期(强烈推荐 JS 直接设置 input value)

翻月点日交互很容易出错。直接给 `.ant-picker-input > input` 写值并触发 React 能识别的 change:

```
renliwo_browser
action: evaluate
code: (function() {
  var inputs = document.querySelectorAll('.ant-picker-input > input');
  if (inputs.length >= 2) {
    var setter = Object.getOwnPropertyDescriptor(window.HTMLInputElement.prototype, 'value').set;
    setter.call(inputs[0], '<起始日期 YYYY-MM-DD>');
    inputs[0].dispatchEvent(new Event('input', {bubbles: true}));
    inputs[0].dispatchEvent(new Event('change', {bubbles: true}));
    setter.call(inputs[1], '<结束日期 YYYY-MM-DD>');
    inputs[1].dispatchEvent(new Event('input', {bubbles: true}));
    inputs[1].dispatchEvent(new Event('change', {bubbles: true}));
    return 'Dates set';
  }
  return 'Inputs not found';
})()
```

⚠️ 这条 JS 改的是受控 input 的 native value,需要用 `Object.getOwnPropertyDescriptor` 拿到 setter 再调用,直接 `inputs[0].value = ...` React 不会更新内部 state。

### 5.6 查询

```
renliwo_browser

```

通过 `pagination` 字段确认筛选生效(条数从全量降到目标范围)。

## 六、坑点速查表

| 现象 | 原因 | 处理 |
|------|------|------|
| 用 `export` 导出"办理信息" → 返回 `async_export: true` | 该接口返回 blob,不触发 download 事件 | 改用 `export_hook_curl` 或本文档第三节手动流程 |
| Ant Tab 点不动 | React 监听的是合成事件,Playwright click 部分场景不响应 | 改 `evaluate` + `dispatchEvent(new MouseEvent('click'))` |
| 福利所在地下拉点不到目标节点 | TreeSelect 虚拟列表,未滚动到的节点不在 DOM 里 | 先 `scrollTop = 150` 再延时 click |
| 日期翻月点错面板 | 左右面板各有独立的 prev/next 按钮 | 改用"JS 直写 input value + change 事件"方案 |
| ZIP 解压后中文乱码 | 文件名 GBK 编码 | 用 Python `name.encode('cp437').decode('gbk')` |
| curl 下载返回 401 | token 失效 | 让用户重登,重新执行 hook 抓 token |
| 福利所在地刚选完就被清空 | 焦点切换太快 | 选完等 2 秒再操作下一个字段 |
| 弹窗残留挡住其他按钮 | 上一次"导出办理信息"没走完 | `evaluate` 点 `.ant-modal-close` 关闭 |
| snapshot 返回空 | 页面还在加载 | wait 2 秒重试 |

---

## 七、推荐操作顺序(批量导出场景)

1. 设好筛选条件 + 查询 → 数据出来后才能导出
2. **先导参保信息**(简单,直接 download):`action='export'`
3. **再导办理信息**(复杂,blob):`action='export_hook_curl'`
4. 解压 ZIP(参保信息那个)
5. 检查两个文件大小,size=0 视为失败需排查

如果先点办理信息没走完,弹窗会遮挡参保信息按钮 → 先 `evaluate` 点 `.ant-modal-close` 再重来。
